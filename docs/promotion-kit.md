## 先统一一句话

把手机或电脑里的录音上传到飞书云盘，Mac 或 Windows 自动本地转写，分别生成智能纪要和完整文字稿，并把两篇文档与完成通知送回飞书。

更短版本：

> 一个把“录音 → 本地转写 → 智能纪要 + 文字稿 → 飞书通知”连起来的开源工作台。

## 为什么值得分享

传播时优先讲用户收益，不要先讲 Python、模型或脚本：

1. **省重复劳动**：录完上传，不再逐条下载、转写、复制、整理和归档；
2. **适合大量长录音**：本地开源转写不消耗飞书妙记分钟数；
3. **拿到的是两篇工作文档**：智能纪要用于快速浏览，完整文字稿用于核对、检索和引用；
4. **默认能直接用，也能适配不同工作**：成熟通用模板为默认，另有客户、采访、播客、课程、培训、项目、调研、复盘和口述场景；
5. **关机后还能继续**：任务按阶段保存，电脑重新开机后继续处理；
6. **设备不受限**：iPhone、Android、微信下载音频和电脑文件都能进入同一 inbox；
7. **隐私和成本更可控**：音频在本机转写，AI 纪要是否调用外部 API 由用户决定；
8. **不是只给程序员**：macOS、Windows 10/11 都有教程，也可以让 AI Agent 代为安装。

模型不绑定单一服务。默认推荐 DeepSeek V4 Flash，当前按 token 计费很低，常见纪要调用成本通常可以忽略；有其他 OpenAI 兼容 API Key 的用户可以直接替换配置。宣传时仍要写“成本通常很低”，不要写“永久免费”或“绝对零成本”。

## 适合谁

- 每月有大量会议、访谈、课程或客户沟通录音；
- 飞书妙记额度不够，希望继续使用飞书归档和协作；
- 希望按自己的行业术语和结构生成纪要；
- 有一台处理时可以保持开机的 Mac 或 Windows 电脑；
- 愿意进行一次初始配置，换取后续长期自动处理。

不适合：偶尔只转一两条、完全不使用飞书、或者不想维护任何电脑端服务的人。对这些用户，飞书妙记等原生服务通常更省事。

## 评论区常见问题

**“是不是完全免费？”**

本地转写不按分钟收费；电脑运行和可选的 AI 纪要 API 仍可能有成本。

**“电脑关机怎么办？”**

录音继续留在飞书 inbox；电脑开机后自动接手。电脑关机时，本地程序无法主动发离线消息。

**“转写准确率多少？”**

准确率受收音、噪声、口音、说话重叠和设备影响，不给统一百分比。仓库提供简体中文实跑样例，也支持后续自定义术语和纪要提示词。

**“为什么不用飞书妙记？”**

少量录音优先用妙记。本项目适合大量长录音、本地转写和自定义纪要结构。

**“Android/Windows 可以吗？”**

Windows 10/11 有安装路径；Android 负责把录音上传到同一个飞书 inbox。真实设备验证状态以验证矩阵为准。
