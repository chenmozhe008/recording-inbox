# 推广素材

以下文案用于发布前准备，不会自动发送到任何平台。

## 一句话介绍

把录音放进飞书文件夹，Mac 或 Windows 自动本地转写、生成智能纪要，并把结果和通知送回飞书。

## GitHub 分享文案

我把自己每天使用的录音整理链路做成了开源精简版：支持 macOS 和 Windows 10/11，iPhone、Android、电脑音频都能进入同一个飞书 inbox；本地 FunASR 转写，不按分钟收费；纪要支持会议、访谈、课程和项目模板。仓库提供配置向导、自检、断电续跑和跨平台 CI。

项目地址：https://github.com/chenmozhe008/recording-inbox

## 小红书短文案

录音多到飞书妙记额度不够，我做了一个开源录音工作台。

手机或电脑把音频传进飞书文件夹，Mac/Windows 自动本地转写，再生成带待办和章节的智能纪要，完成后飞书直接提醒。转写不按分钟收费，纪要模板还能自己改。

适合会议、采访、课程和客户沟通很多的人。偶尔转几条，直接用妙记更省事。

项目地址：https://github.com/chenmozhe008/recording-inbox

## 演示素材清单

- 15 秒：音频拖入飞书 inbox；
- 20 秒：终端配置向导和全绿自检；
- 15 秒：飞书收到完成消息并打开纪要；
- 1 张：README 流程图；
- 1 张：示例纪要的智能摘要、待办和文字记录。

录屏前使用测试账号和脱敏音频，画面中不得出现 Key、Webhook、open_id 或真实客户内容。
