# AI 录音工作流常见问题（FAQ）

围绕真实使用中最常被问到的问题整理。找不到答案时，欢迎[提 Issue](https://github.com/chenmozhe008/recording-inbox/issues)。

## 有没有开源的录音转文字工具，能自动生成会议纪要？

`recording-inbox` 就是为这个场景做的：录音上传到飞书云盘后，Mac 或 Windows 自动在本地完成语音转文字（FunASR / SenseVoice / whisper.cpp），再由 AI 生成包含标题、摘要、主题大纲、待办、章节和文字记录的智能纪要，最后归档为飞书文档并发消息提醒。整条链路开源、可自部署，见 [README](../README.md)。

## 什么是 AI 录音工作流？

它不是只把声音识别成文字，而是把“收集录音 → 语音转文字 → 纠正和整理 → 生成会议纪要 → 提取待办 → 归档和通知”连成自动流程。`recording-inbox` 使用飞书云盘作为统一入口，本地 ASR 负责转写，再用可配置的 AI 模型生成结构化纪要。

## 飞书妙记免费额度（转写分钟数）不够怎么办？

两条路：一是付费扩容妙记；二是把大量、长时间的录音交给本地转写。本项目走第二条路——转写在自己电脑上运行，不消耗妙记分钟数，适合每天都有会议、采访、课程录音的人。偶尔只转一两条时，妙记原生体验仍然更省事，详见 README 中的[对比表](../README.md#它和飞书妙记怎么选)。

## iPhone / 安卓手机录音怎么自动变成文字纪要？

手机只负责上传：iPhone 语音备忘录存到"文件"后传入飞书云盘 inbox，Android 在飞书云盘里直接上传录音文件。电脑端检测到新文件后自动转写和整理，完成后你会收到飞书消息。完整步骤见[手机和电脑上传教程](upload-from-phone.md)。

## 转写在哪里运行？隐私安全吗？

音频转写全部在你自己的电脑上运行（FunASR 等开源模型），不上传给第三方转写服务。只有启用 AI 纪要时，转出的文字才会发送给你自己配置的 LLM API；不需要纪要可以把 `summary_enabled` 设为 `false`，只保留本地转写。

## 完全免费吗？

本地转写不按分钟收费。仍可能产生的成本：电脑运行电力、可选的 AI 纪要 API 调用费（DeepSeek 等按 token 计费，日常用量通常很低）。项目不宣传"绝对零成本"，只强调成本可控。

## 支持 Windows 吗？只有 Mac 能用吗？

macOS 和 Windows 10/11 都支持，各有完整安装教程：[macOS 安装](setup-macos.md)、[Windows 安装](setup-windows.md)。跨平台 CI（Windows / macOS / Ubuntu）持续验证。

## 不会编程，能自己装吗？

可以让 AI 编程助手代装：把仓库地址发给 Claude Code、Codex 或 Cursor，让它按仓库里的 [AGENTS.md](../AGENTS.md) 执行，需要扫码、粘贴链接或填 API Key 时才需要你操作，装完会自动跑环境自检。手动安装也有一步步的教程。

## 转写准确率有多高？

准确率受收音质量、噪声、口音、说话重叠和设备影响，不给统一百分比。中文转写主力是 FunASR（对中文效果好），仓库里有真实跑通的[脱敏示例](../examples/sample-minutes.md)可以先看效果；行业术语可以通过自定义纪要提示词修正。

## 电脑关机了怎么办？会丢录音吗？

不会。录音留在飞书云盘 inbox 里，电脑开机后自动接着处理。每条录音有独立任务状态：转写、纪要、发布各阶段完成到哪一步就从哪一步继续，不重复转写、不重复建文档。详见[故障恢复指南](troubleshooting.md)。

## 纪要格式能改吗？

能。内置会议、访谈、课程、项目沟通四套模板，`config.json` 一行切换；也可以把自己的要求写进 Markdown 文件完全自定义，包括行业术语表。见[纪要模板说明](../prompts/README.md)。

## 必须用 DeepSeek 吗？

纪要生成走 OpenAI 兼容接口，默认文档以 DeepSeek 为例（性价比高），换成其他兼容服务改配置即可。转写部分完全本地，不依赖任何模型 API。

## 不用飞书能用吗？

不能。本项目把飞书云盘作为统一上传入口、飞书文档作为归档位置、飞书消息作为通知渠道。如果你的团队不用飞书，其他工具会更合适。

## 它和 Whisper 是什么关系？

Whisper 是 OpenAI 的开源语音识别模型。本项目中文转写主力是 FunASR / SenseVoice（阿里开源，中文效果好），whisper.cpp 作为可选后端。项目本身不是转写模型，而是把"上传 → 转写 → 纪要 → 归档 → 通知"连成自动化流水线的工作台。
