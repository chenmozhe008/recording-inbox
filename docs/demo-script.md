# 演示录屏脚本

目标不是展示终端有多复杂，而是让第一次看到项目的人在一分钟内理解：录音从哪里来、系统自动做了什么、最后得到什么。

## 素材准备

- 一条 20-40 秒的脱敏测试录音；
- 飞书云盘中的测试 `inbox` 和输出文件夹；
- 已完成配置并通过 `scripts/setup_check.py` 的电脑；
- 一篇由测试录音生成的智能纪要；
- 关闭飞书消息预览和无关通知；
- 清空终端中可能出现的 Key、token、账号和本机绝对路径。

建议测试录音内容：

```text
今天确认三件事。第一，周三前确定拍摄场地。第二，明天中午前反馈剪辑意见。第三，设备采购已经提交审批。
```

## 30 秒短版

适合小红书、视频号、朋友圈和仓库首页短视频。

| 时间 | 画面 | 字幕 |
|---|---|---|
| 0-3 秒 | 手机录音或电脑上的音频文件 | 录音容易，整理最费时间 |
| 3-7 秒 | 音频进入飞书云盘 `inbox` | 只要把录音上传到一个文件夹 |
| 7-12 秒 | 电脑自动接手的状态或简洁日志 | Mac / Windows 自动本地转写 |
| 12-20 秒 | 脱敏智能纪要，从摘要滑到待办 | 自动生成摘要、主题、待办和文字记录 |
| 20-25 秒 | 飞书完成通知，点击进入文档 | 完成后飞书直接提醒 |
| 25-30 秒 | GitHub README 和项目地址 | 开源：recording-inbox |

旁白：

```text
把手机或电脑录音上传到飞书，Mac 或 Windows 会自动在本地转写，再生成带摘要、待办和文字记录的智能纪要，完成后直接在飞书提醒。这个开源项目叫 recording-inbox。
```

## 60 秒完整版

适合 B 站、知乎、小红书长视频和技术社区。

| 时间 | 画面 | 说明 |
|---|---|---|
| 0-5 秒 | 录音列表和一份未整理文件 | 提出问题：真正耗时的是录完后的整理 |
| 5-12 秒 | 手机或电脑上传到 `inbox` | 展示统一入口，不快速跳过关键按钮 |
| 12-18 秒 | `setup_check.py` 全绿结果 | 说明第一次安装后会自动运行 |
| 18-25 秒 | 任务从待处理到转写、纪要、发布 | 不展示长日志，只展示可理解状态 |
| 25-38 秒 | 智能纪要标题、总览、主题和待办 | 强调不是只输出逐字稿 |
| 38-45 秒 | 文字记录和单人/多人标签效果 | 单人不显示无意义的“说话人1” |
| 45-51 秒 | 飞书完成通知 | 说明不用一直盯处理进度 |
| 51-57 秒 | README 中模板、恢复和隐私部分 | 强调可改模板、关机续跑、本地转写 |
| 57-60 秒 | 仓库地址和 Star 提示 | 给出唯一行动入口 |

旁白：

```text
开会、采访和客户沟通录完以后，还要转写、整理重点、提取待办、重新命名和归档。我把这些步骤连成了一个开源工作流。

手机或电脑把音频上传到飞书 inbox，Mac 或 Windows 自动用 FunASR 在本地转写，再按会议、访谈、课程或项目模板生成智能纪要。结果回到飞书，完成后直接收到提醒。

它适合长录音多、妙记额度不够、又希望纪要格式能自己控制的人。项目包含安装向导、自检、断电续跑和跨平台测试，地址在画面上。
```

## iPhone 上传录屏

按 [移动端真机录屏清单](mobile-demo-checklist.md) 执行。画面必须完整出现：

```text
语音备忘录
→ 存储到文件
→ 飞书云盘
→ inbox
→ 右下角上传
→ 选择文件
→ 文件出现在 inbox
```

不要只录“从收藏进入”，否则第一次使用的人看不懂入口在哪里。

## Android 上传录屏

画面必须完整出现：

```text
飞书云盘
→ inbox
→ 右下角上传
→ 系统文件管理器
→ 选择录音
→ 文件出现在 inbox
```

不同品牌的文件管理器界面不同，以真实设备为准，不用模拟画面替代。

## 电脑拖拽演示

这是最容易录、也最适合先发布的一条：

1. 打开飞书云盘 `inbox`；
2. 把一条测试音频拖进去；
3. 显示文件上传完成；
4. 切到电脑任务状态；
5. 展示输出文档和飞书完成通知。

可以加速等待片段，但必须在字幕中写“等待过程已加速”，不要制造实时完成的错觉。

## GitHub 截图顺序

1. README 第一屏：一句话定位、平台徽章和核心优势；
2. 30 秒流程图；
3. 脱敏智能纪要截图；
4. 安装向导或全绿自检；
5. Windows/macOS/Ubuntu CI；
6. 验证矩阵中的“已验证”和“待人工验证”。

## 录制验收

- 手机竖屏素材保持 9:16，桌面演示保持 16:9；
- 关键按钮和文件出现的一刻不能被剪掉；
- 字幕不遮挡按钮、待办和项目地址；
- 不展示真实客户、联系人、内部录音和通知预览；
- 不展示 `.env`、`config.json`、token、open_id、Webhook 和个人绝对路径；
- 不写“永久免费”“100% 准确”“固定几分钟完成”；
- 结尾只保留一个明确入口：GitHub 仓库地址。
