# 上传录音：iPhone、Android 和电脑

所有入口最终都进入同一个飞书 inbox。第一次使用后，把 inbox 加入飞书“收藏”。

> 飞书移动端会随版本更新调整按钮名称和位置。本文给出的是稳定的操作逻辑；
> 发布教程时应使用当前版本真机录屏核对界面，不使用 iOS Simulator 里的 Mac 版飞书外壳冒充真机验证。

不同版本的入口可能显示为“云文档”或“云盘”，常用文件夹可能显示在“收藏”或“快速访问”。它们都是正常的版本差异。上传录音时选择“上传文件”，不要选择“导入为在线文档”。

## 电脑拖拽：最简单

适合微信、网盘、聊天软件或已经同步到电脑的音频：

1. 打开飞书云盘中的 inbox；
2. 把音频拖进窗口；
3. 等电脑下一轮自动接手。

支持 `.wav`、`.mp3`、`.m4a`、`.aac`、`.mp4`、`.mov`。

## iPhone

系统“语音备忘录 → 分享 → 飞书”在部分版本只提供聊天或飞书妙记，不一定能保存到云盘 inbox。最稳定的通用路径是：

1. 语音备忘录录完，点分享；
2. 选择“存储到文件”；
3. 打开飞书 App，进入已收藏的 inbox；
4. 点 `+` → 上传 → 从“文件”选择刚才的录音。

第一次需要多一步“存储到文件”，以后从收藏进入 inbox 会快很多。

## Android

不同品牌的系统录音机分享菜单差异很大。通用路径：

1. 打开飞书 App；
2. 从“收藏”进入 inbox；
3. 点 `+` → 上传；
4. 从文件管理器选择录音。

如果文件管理器的分享菜单明确出现“保存到云盘”，也可以使用；不要选择“音视频转文字”，那会走飞书妙记并消耗对应额度。

## iPhone 快捷指令（高级）

适合每天大量录音、愿意维护飞书自建应用的人。它不是首次安装必选项。

大致流程：

1. 飞书开放平台创建自建应用并开启文件上传权限；
2. inbox 文件夹共享给应用；
3. 快捷指令接收语音备忘录分享输入；
4. 获取 tenant access token；
5. 调用飞书文件上传 API 写入 inbox；
6. 本地显示上传成功。

App Secret 会保存在快捷指令中，不要把该快捷指令公开分享。由于飞书开放平台权限和快捷指令界面会变化，这条路径需要在自己的真机上验证；失败时退回飞书 App 手动上传，不影响主流程。

## 电脑没开机怎么办？

- 上传仍然成功，录音留在飞书 inbox；
- 电脑开机后自动下载和处理；
- 电脑关机期间本地程序无法发送“电脑离线”消息；
- 急用时可以临时用飞书妙记处理这一条。

## 如何判断上传成功？

在飞书 inbox 能看到音频即可。电脑在线时，通常下一分钟接手；处理后的原音频会移进 inbox 的 `processed` 子文件夹。

## 教程素材验证状态

- 电脑拖拽、后台拉取、断点续跑和结果发布由自动测试覆盖；
- iPhone 与 Android 的按钮位置必须以当前飞书真机界面为准；
- 录制或更新教程时，按 [移动端真机录屏清单](mobile-demo-checklist.md) 一次走完，避免只截到上传前半段。
