# Changelog

## Unreleased

- 修复转写清洗误删句末逗号、句号的问题，保留 ASR 原有断句信息并增加回归测试。
- 新安装统一使用经过实跑验证的 FunASR 1.3.10 / ModelScope 1.37.1 核心基线，避免依赖升级静默改变转写结果。
- 移除工作流未使用且容易与 PyTorch 版本错配的 torchaudio 安装项。
- 环境自检会显示实际 ASR 版本；偏离验证基线时给出升级提示，但不阻断已有可用环境。
- 补充首次模型下载与本机缓存说明，macOS 和 Windows 安装命令改用平台依赖文件。
- 默认模型更新为 DeepSeek V4 Pro（纪要质量更好，单次调用通常只有几分钱），可在配置中改回 `deepseek-v4-flash`。
- 后台扫描间隔从每 1 分钟调整为每 3 分钟（launchd 与 Windows 任务计划模板同步更新），已安装用户重装定时任务即可生效。
- 飞书默认文件夹统一为“录音收件箱”和“录音结果”；原音频接收后归档到“已接收原始音频”。
- 保留旧 `processed` 原始音频目录的兼容性，已有部署无需迁移即可继续运行。

## 0.2.0 - 2026-07-12

- 每条录音分别生成“智能纪要”和“文字稿”两篇飞书文档，并通过一条消息提供两个入口。
- 增加 10 个内置纪要模板，默认使用通用会议纪要模板。
- 默认模型更新为 DeepSeek V4 Flash，并支持 OpenAI Chat Completions 兼容服务。
- 完成真实双文档发布与完成通知验收。

## 0.1.0 - 2026-07-11

- 默认使用已授权的 lark-cli 给当前飞书账号发送结果消息，Webhook 下沉为兼容选项。
- 增加 macOS / Windows 统一配置向导和通知权限自检。
- 修复电脑断电后遗留锁可能阻塞数小时的问题。
- 已发布纪要通知失败时只补通知，不重复创建文档。
- 增加会议、访谈、课程、项目沟通模板和自定义提示词入口。
- 增加模拟端到端、敏感信息、文档链接和跨平台 CI 检查。
- 重构安装、上传、API、安全、恢复、示例和推广文档。
- 恢复 GitHub 首页的核心优势、使用场景、产品对比和可信验证证据。
- 增加多渠道推广文案、演示录屏脚本、发布节奏和传播红线。
